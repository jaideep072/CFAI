Explore the visualizer workspaces, algorithms, and models powering the Cognitive Path Solver.

1. Pathfinding & CSP Workspace

Visualize uninformed, heuristic-guided search, and constraint satisfaction solvers on dynamic terrain grids.

Supported Search Algorithms:
  • BFS & Bidirectional BFS: Gurantee shortest path; Bidirectional BFS searches from both ends to reduce node expansion.
  • Uniform Cost Search (UCS): Finds the cheapest path considering terrain weights (Mud = 3, Water = 6, Open = 1).
  • Depth-First Search (DFS): Fast, memory-efficient search; does not guarantee optimal paths.
  • A* & Weighted A*: Heuristic search utilizing Manhattan, Euclidean, or Chebyshev distance. Weighted A* scales heuristics by $w$ (1.0 - 5.0) to speed up search.
  • IDA* (Iterative Deepening A*): Depth-first search with heuristic cost cutoffs, combining DFS memory efficiency with A* optimality.
Heuristic Functions:
  • Manhattan Distance: Horizontal + vertical step count (admissible on 4-way grids).
  • Euclidean Distance: Straight-line vector distance (always admissible).
  • Chebyshev / Diagonal: Maximum of horizontal and vertical changes.
  • Zero Heuristic: Equivalent to UCS ($h=0$).
Constraint Satisfaction (CSP) Tools:
  • Path 3-Coloring: Solves graph coloring constraints using backtracking search with MRV, LCV, and Forward Checking to assign colors to path steps.
  • Waypoint Scheduling: Schedules 5 checkpoints along the path into chronological slots ($T_1 < T_2 < \dots < T_5$).
Comparative Benchmark Table:

Runs all algorithms on the active grid layout and displays path length, nodes expanded, memory peak, and CPU latency to compare efficiency.

How to Use:
  1. Switch to **Pathfinding & CSP** in the navigation header.
  2. Draw walls or weight cells (Mud/Water) on the grid, or choose a preset.
  3. Choose an algorithm and click **Solve & Animate** to start the visualization.
  4. Once solved, run Path Coloring or Waypoint Scheduling from the right panel.

2. Adversarial Arena

Simulate zero-sum multi-agent games modeling an Evader (Agent) and a Chaser (Ghost) on the grid.

Decision Strategies:
  • Minimax (with Alpha-Beta Pruning): Assuming a worst-case optimal adversary. Prunes branches to speed up search.
  • Expectimax Search: Models the ghost as a stochastic agent, calculating expected utility over random ghost moves instead of absolute minimax.
  • Search Depth: Slider adjusting lookahead depth (from 2 to 6 steps).
How to Use:
  1. Switch to **Adversarial Arena** tab.
  2. Click **Place Ghost Mode** and select any open grid cell to position the ghost.
  3. Select your Evader Strategy (Minimax or Expectimax) and depth.
  4. Use **Step** to play single turns, or **Auto-Play** to run the game loop.

3. Probabilistic HMM Tracker

Track a hidden ghost moving stochastically using noisy distance readings and probabilistic inference.

Core Inference Models:
  • Hidden Markov Model (HMM) Filter: Calculates a probability distribution heatmap over all open grid cells. Updates belief at each step using transition probability and sensor likelihood.
  • Variable Elimination: Computes exact marginal queries (e.g., $P(\text{Ghost Near} \mid \text{Alarm Active})$) by eliminating factor variables algebraically in a Bayes Net.
How to Use:
  1. Switch to the **Probabilistic Tracker** tab.
  2. Place a ghost on the grid in Adversarial Arena.
  3. Click **Update HMM** to capture a noisy sensor reading and see the probability distribution heatmap update.
  4. Click **Run Worked Trace** to view the step-by-step mathematical trace of Variable Elimination.

4. Hybrid Reasoner

Evaluate paths under uncertainty by combining threat belief maps, risk tolerance, and CSP scheduling constraints.

Decision Theory & CSP:
  • Expected Utility (EU) Decision: Compares the direct path (shorter but passes near the ghost) against a detour path. Calculates $EU = \sum P(\text{safe}) \cdot U(\text{cost}) + P(\text{threat}) \cdot U(\text{capture})$.
  • Risk Tolerance ($w$): A slider adjusting preference from risk-averse ($w=0.0$, detours preferred) to risk-tolerant ($w=1.0$, direct paths preferred).
How to Use:
  1. Switch to the **Hybrid Reasoner** tab.
  2. Place a ghost, and run a few HMM updates to build a localized threat probability map.
  3. Adjust the **Risk Tolerance** slider on the left controls.
  4. Click **Execute Decision Cycle** to compute expected utilities and print the reasoning trace.

PEAS Environment Profile

Performance Measures
  • Path Cost: Accumulation of terrain step weights (Open=1, Mud=3, Water=6).
  • Search Space: Nodes expanded during pathfinding frontier propagation.
  • Computational Cost: Latency in milliseconds to execute solvers.
Environment
  • Fully observable grid edges, wall layouts, and traversal costs.
  • Episodic, static obstacles, and deterministic actions.
Actuators

Standard grid actions: displace North (N), South (S), East (E), and West (W).

Sensors

Location coordinate tracking, obstacle checks, and noisy distance sensors.
